# 09 · Search page (React + Inertia)

> **Goal:** a page at `/bible` with a search box and the list of results (reference, text and book).

> 💡 In Claude Code, activate the **`inertia-react-development`** skill when writing this code to
> follow the correct Inertia v3 patterns. Reuse existing components from
> `resources/js/components/` before creating new ones.

## Step 1 · Create the page

Create `resources/js/pages/bible/search.tsx`. It must match the name used in
`Inertia::render('bible/search', ...)` from guide 08.

```tsx
import { Head, Form } from '@inertiajs/react';
import { search as bibleSearch } from '@/routes'; // route generated by Wayfinder (adjust the import)

interface Result {
    id: number;
    reference: string;
    text: string;
    book: string;
}

interface Props {
    query: string;
    results: Result[];
}

export default function BibleSearch({ query, results }: Props) {
    return (
        <>
            <Head title="Bible search" />

            <div className="mx-auto max-w-2xl px-4 py-10">
                <h1 className="mb-6 text-2xl font-semibold">Semantic Bible search</h1>

                <Form action={bibleSearch()} method="get" className="mb-8 flex gap-2">
                    {({ processing }) => (
                        <>
                            <input
                                type="text"
                                name="q"
                                defaultValue={query}
                                placeholder="e.g. love your neighbor"
                                className="flex-1 rounded-md border px-3 py-2"
                            />
                            <button
                                type="submit"
                                disabled={processing}
                                className="rounded-md bg-black px-4 py-2 text-white disabled:opacity-50"
                            >
                                {processing ? 'Searching…' : 'Search'}
                            </button>
                        </>
                    )}
                </Form>

                {query !== '' && results.length === 0 && (
                    <p className="text-gray-500">No verses found for “{query}”.</p>
                )}

                <ul className="space-y-4">
                    {results.map((r) => (
                        <li key={r.id} className="rounded-md border p-4">
                            <p className="mb-1 text-sm font-medium text-gray-500">{r.reference}</p>
                            <p>{r.text}</p>
                        </li>
                    ))}
                </ul>
            </div>
        </>
    );
}
```

> **The route import**: after `php artisan wayfinder:generate` (guide 08), confirm the real import
> path. It's usually from `@/routes` (named routes) or `@/actions/...` (by controller). Adjust
> `import { search as bibleSearch } from '...'` to whatever got generated. If you'd rather not use
> Wayfinder yet, you can use `action="/bible" method="get"` directly.

## Step 2 · Build the frontend

In development:

```bash
npm run dev
```

Or the project's combined flow:

```bash
composer run dev
```

## Step 3 · Open the page

Go to the project URL + `/bible` (e.g. `http://bible.test/bible` with Herd, or
`http://127.0.0.1:8000/bible` with `php artisan serve`). Type a meaning-based search.

---

## Optional improvement: loading state

Since the query generates an embedding with Ollama, it can take ~1s. The `<Form>`'s `processing`
already disables the button; if you want, show a *skeleton* while `processing` is `true`.

## How to verify

- [ ] `/bible` loads with no errors in the browser console.

- [ ] Results show reference and text.
